# 会话交接（Handoff）

## 核心原则

**交接文档不是会话记录**。不要把每一轮对话都写进去。交接文档是压缩后的"当前状态快照 + 下一步行动清单"。

---

## 交接流程

### 第一步：盘点已有产出

先列出本次会话已经产出的持久化内容，它们在交接文档中只需引用路径/URL，不需要重复：

- PRD → 如果用了 `/to-prd`，引用 issue URL
- ADR → 引用 `docs/adr/` 中的文件
- Issue → 引用 GitHub issue 编号或 URL
- Commit → 引用 commit SHA
- PR → 引用 PR URL
- 已写入的文件 → 引用文件路径

### 第二步：提取未持久化的关键信息

以下内容**尚未**写入持久化文件，必须包含在交接文档中：

1. **当前在做什么**：正在实现哪个功能/修哪个 bug？进行到哪一步了？
2. **关键决策**：做了哪些设计选择？为什么？
3. **未完成的工作**：还剩什么要做？按优先级排列
4. **已知陷阱**：踩过什么坑？有哪些该避免的方向？
5. **当前分支/工作树状态**：在哪个分支？有没有未提交的修改？

### 第三步：脱敏

**必须移除**以下内容：
- API key、token、密码
- aifriend 测试账号密码（`773682014@qq.com` / `jie159357` 是公开测试账号，不需要脱敏，可以在交接文档中引用）
- 生产环境连接信息
- 用户隐私数据
- 内部 IP 地址

### 第四步：标注建议技能

列出下一个 Agent 应该加载的技能，帮助它快速进入状态。例如：
- 继续写代码 → 建议加载 `tdd`
- 排查问题 → 建议加载 `diagnose`
- 代码审查 → 建议加载 `improve-codebase-architecture`
- 需求讨论 → 建议加载 `grill-me` 或 `grill-with-docs`

### 第五步：输出

将交接文档写入 `.scratch/handoff-YYYY-MM-DD-HHMM.md`（项目 .scratch/ 目录，不提交到 git）。

## 交接文档模板

```markdown
# 会话交接 — [简短标题]

**日期**：YYYY-MM-DD HH:MM
**分支**：[分支名]
**未提交修改**：有/无（如果有，列出关键文件）

## 当前进度
[一段话描述正在做什么，做到哪一步了]

## 关键决策
- 决策 1 + 一行理由
- 决策 2 + 一行理由

## 待完成
- [ ] 任务 1（优先级：高/中/低）
- [ ] 任务 2
- [ ] 任务 3

## 已知陷阱
- 陷阱 1
- 陷阱 2

## 建议技能
- `skill-1` — 原因
- `skill-2` — 原因

## 引用
- 相关 issue：[链接]
- 相关 PR：[链接]
- 相关文件：[路径列表]
```

---

## 质量检查

好的交接文档满足：
1. 下一个 Agent 读完能在 1 分钟内理解当前状态
2. 不包含任何可以通过路径/URL 查到的重复内容
3. 不包含任何敏感信息
4. 下一步行动有明确优先级
